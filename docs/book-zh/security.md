# 安全与隐私

## 什么会离开你的机器

只有你选择发送到已配置服务商的请求。Token Guard 将字节转发给上游 API，并在本地记录元数据。

## 记录了什么

`logs` 表存储：

- 时间戳
- 服务商名称
- 模型名称
- 提示令牌数
- 补全令牌数
- 估算成本
- 请求时长
- 项目标签
- HTTP 状态码

除非显式启用请求/响应体日志，否则不会记录提示词、补全内容、嵌入向量或其他请求/响应体。

## API 密钥存储位置

服务商 API 密钥存储在系统钥匙串中：

- Windows：凭据管理器
- macOS：钥匙串
- Linux：D-Bus Secret Service（GNOME Keyring、KeePassXC、KWallet）

密钥永远不会写入 SQLite 数据库、`settings.json` 或任何配置文件。

## 项目标签密钥

你在代理中设置为 `OPENAI_API_KEY` 的值是**标签密钥**，不是秘密。它只用于将请求映射到项目。即使别人看到，也无法访问你的服务商账户。

如果请求携带的标签密钥与任何项目不匹配，Token Guard 会在转发前以 HTTP 401 拒绝。

## 网络绑定

默认情况下代理绑定到 `127.0.0.1`，因此网络上的其他设备无法访问。局域网暴露是可选的。

## 更新

Token Guard 会检查 GitHub Releases 更新。更新检查不会发送使用数据、提示词或遥测。

## 许可证验证

许可证密钥通过 Cloudflare Worker 验证。Worker 仅接收密钥和设备指纹；不会接收提示词、使用数据或服务商密钥。
